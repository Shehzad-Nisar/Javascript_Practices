JavaScript Synchronous vs Asynchronous

1️- Synchronous JavaScript

Synchronous code runs line by line. Each task must finish before the next task starts.

console.log("Start"); console.log("Task 1"); console.log("Task 2"); console.log("End");
Output:
Start
Task 1
Task 2
End

2️- Asynchronous JavaScript

Asynchronous code allows some tasks to run later without blocking the rest of the program.

console.log("Start"); setTimeout(function(){ console.log("Task after 2 seconds"); },2000); console.log("End");
Output:
Start
End
Task after 2 seconds

3️- What is setTimeout?

setTimeout executes a function after a specific delay.

setTimeout(function(){ console.log("Hello after 3 seconds"); },3000);

4️- Callback Hell

Callback Hell happens when many asynchronous operations are nested inside each other. This makes the code difficult to read and maintain.

setTimeout(function(){ console.log("Step 1"); setTimeout(function(){ console.log("Step 2"); setTimeout(function(){ console.log("Step 3"); setTimeout(function(){ console.log("Step 4"); },1000); },1000); },1000); },1000);

This pyramid-shaped structure is called Callback Hell.
